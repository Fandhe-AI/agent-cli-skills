# 日本語スタイル規約

## 基本方針

全スキル・Agent・Rule の**出力・レポート・エラーメッセージは日本語**で記述する。

## 言語の使い分け

| 要素 | 言語 |
|-----|------|
| ユーザーへの説明・報告・エラーメッセージ | 日本語 |
| コード・コマンド・識別子（変数名・関数名・ファイル名） | 英語のまま |
| コミットメッセージの subject・body | 日本語可（type・scope は英語） |
| PR タイトル・Issue タイトル | 日本語可（Conventional Commits の type は英語） |
| PR / Issue の本文 | 日本語可 |
| コード内コメント | 日本語可 |
| YAML frontmatter の値 | 日本語可（キー名は英語） |

例:

```
feat(auth): ソーシャルログイン機能を追加   # subject は日本語可
```

```bash
# コマンドは英語のまま
gh pr create --title "feat(auth): ソーシャルログイン機能を追加"
```

## トーン

- **常体（だ・である調）** を基本とする
- ユーザーへの案内・確認は常体で簡潔に
- レポート・一覧は常体
- 丁寧語（「ください」「します」）は必要最小限に留める

## 禁止事項

- 日本語のメッセージに英語のみで返答しない
- コード・コマンドを日本語に翻訳しない（識別子・コマンド名は原文を維持）
- 絵文字の多用（レポートの判定記号 ✅/⚠️/❌ は許可）
